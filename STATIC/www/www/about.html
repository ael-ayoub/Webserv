THE PROJECT

A FULLY FUNCTIONAL HTTP/1.1 SERVER BUILT FROM SCRATCH IN C++98

This project implements a complete web server following the HTTP/1.1 protocol specification. It handles multiple simultaneous connections, serves static files, and executes CGI scripts for dynamic content generation.

IMPLEMENTATION DETAILS

01

SOCKET MANAGEMENT

The server creates listening sockets for each configured port and uses poll() to monitor multiple file descriptors simultaneously. This allows handling many concurrent connections without blocking.

02

REQUEST PARSING

HTTP requests are parsed into method, URI, headers, and body components. The parser handles chunked encoding, multipart form data, and URL-encoded payloads.

03

RESPONSE GENERATION

Responses include proper HTTP headers, status codes, and content. The server determines MIME types, handles redirects, and generates error pages for common HTTP errors.

04

CGI EXECUTION

When a CGI script is requested, the server forks a child process, sets up environment variables, and executes the script. Output is captured via pipe and wrapped in HTTP response format.
